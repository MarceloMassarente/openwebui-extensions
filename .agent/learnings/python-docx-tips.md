# python-docx Metadata Sanitization & Styling Learnings

## Metadata Fingerprints

### 1. Hardcoded `last_modified_by`
By default, `python-docx` (specifically in `docx.opc.parts.coreprops`) hardcodes `last_modified_by = "python-docx"`. Even if you clear `author` and `comments`, this "Last Saved By" field will still reveal the library usage in Word file properties.
**Solution**: Explicitly set `doc.core_properties.last_modified_by` to the desired user name or an empty string.

### 2. Core Properties Mapping
- `doc.core_properties.author` -> `<dc:creator>`
- `doc.core_properties.comments` -> `<dc:description>` (This is where "generated by python-docx" usually resides).
- `doc.core_properties.title` -> `<dc:title>`

## Heading Font Configuration

To ensure headings use specific fonts (like Aptos Display or Calibri Light) while maintaining CJK (Chinese/Japanese/Korean) compatibility:
1. Access the style via `doc.styles["Heading 1"]`.
2. Modify `style.font.name`.
3. **Crucial**: Manually update the `rFonts` element in the underlying XML (`style._element.get_or_add_rPr().get_or_add_rFonts()`).
4. Set `w:ascii` and `w:hAnsi` to the target Latin font.
5. Set `w:eastAsia` to the target Asian font (e.g., `SimSun`) to prevent fallback to generic system fonts like "MS Gothic".
6. Set the language property (`w:lang`) to a specific locale (e.g., `zh-CN`) to avoid layout shifts or incorrect character rendering in some Word versions.
