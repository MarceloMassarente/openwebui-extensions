# 异步上下文压缩 v1.6.2 版本发布说明

## 概述

本补丁版本修复了一个关键 Bug：Outlet 的后台压缩任务在首次压缩后会在每条后续消息上重复触发，浪费 LLM 调用并降低性能。同时改进了 Token 状态显示的准确性，并新增滞后守护机制防止不必要的重复压缩。

## 问题修复

- **修复首次压缩后重复触发问题**（Issue #68）：Outlet 的 Token 阈值检查之前对完整原始消息历史计数，一旦压缩触发过就永远超阈值。现在改为模拟实际发送上下文（head + summary + 保留的系统消息 + tail）进行准确的阈值比较。
- **修复上下文用量显示超过 100%**：状态通知现在报告模拟发送上下文的 Token 数，而非完整历史，避免误导性的用量比例。

## 新功能

- **滞后守护**：压缩触发后，新增守护机制要求至少积累 `keep_last` 条新消息才允许再次触发压缩。避免每轮仅压缩 1-2 条消息的浪费性 LLM 调用。
- **增强调试日志**：阈值检查日志现在显示估算 Token 数、精确计算触发阈值和 tiktoken 结果，便于调试。

## 改进

- **更准确的 tiktoken 触发时机**：精确 Token 计算现在在达到压缩阈值的 60% 时触发（之前为 85%），更早提供准确的上下文用量状态。

## 迁移说明

无破坏性变更。可直接替换 v1.6.1。
