# 异步上下文压缩 v1.6.5 版本发布说明

## 概述

这个版本新增了可配置的摘要压缩风格。管理员现在可以选择摘要更偏向最大 token 节省、默认平衡连续性，或更高保真地保留细微上下文。

## 新功能

- **`compression_style` 配置项**：新增 `aggressive`、`balanced`、`faithful` 三种模式。
- **aggressive 模式**：优先压缩 token，只保留最可能影响下一轮回复的事实。
- **balanced 模式**：保持默认折中策略，保留决定性事实、关键理由、仍在比较的方案、重要限制和未解决问题，同时移除低价值细节。
- **faithful 模式**：更充分使用已配置的摘要预算，保留关键推理链、评估标准、候选方案、重要差异点，以及能说明上下文意义的示例。

## 配置说明

`compression_style` 默认值为 `balanced`，因此现有安装在不修改配置时会保持原有的实际使用体验。

建议：

- 追求最大上下文成本节省时使用 `aggressive`。
- 一般长对话连续性场景使用 `balanced`。
- 编码、规划、分析和评审类对话使用 `faithful`，避免丢失会影响后续回答的细节。

## 迁移说明

无破坏性变更。请更新或重新安装过滤器，确保 OpenWebUI 中保存的 function 内容包含新的配置项元数据和摘要提示词说明。
