# Export to Word Enhanced — v0.5.0 版本发布说明

**发布日期：** 2026-05-03
**作者：** [Fu-Jie](https://github.com/Fu-Jie)
**版本：** 0.5.0

---

## 最新更新

### 🔠 标题级别字体自定义

新增阀门，可为 H1 和 H2 独立配置字体，不再与正文字体绑定：

| 阀门 | 默认值 | 说明 |
| :--- | :--- | :--- |
| `一级标题西文字体` | `Aptos Display` | H1 西文字体 |
| `一级标题中文字体` | `SimHei` | H1 中文字体 |
| `二级标题西文字体` | `Calibri Light` | H2 西文字体 |
| `二级标题中文字体` | `SimHei` | H2 中文字体 |

留空则回退到 `英文字体` / `中文字体`。`export_to_word.py`（英文阀门）和 `export_to_word_cn.py`（中文阀门）已同步更新。

### ⬆️ 相对标题算法

当导出内容没有 `# H1` 时，插件会扫描最小标题级别并自动提升所有标题。例如，如果内容从 `##` 开始，所有 `##` 变为 `#`，`###` 变为 `##`。提升时会输出调试日志。

### 🧹 元数据清理

生成的 Word 文档现在会将 `author`、`last_modified_by`、`title` 等核心属性设置为真实用户名和对话标题，移除了之前泄露到文档属性中的 `python-docx` 库默认指纹。

### 🎨 标题样式修复（关键）

- **主题字体清除**：清除了默认样式、Title 样式和 Heading 1-9 样式上的 `w:asciiTheme`、`w:hAnsiTheme`、`w:eastAsiaTheme` 和 `w:cstheme`。这防止了 Word 用主题默认值覆盖显式字体选择（如宋体回退到 MS Gothic）。
- **Run 级别 CJK 强制**：对于 Heading 段落，东亚字体现在在 Run 级别设置（`w:eastAsia` + 清除 `w:eastAsiaTheme`），解决了仅靠样式级别字体不可靠的问题。
- **移除多余的下划线和边框**：标题样式不再从默认模板继承下划线或段落边框装饰。

---

## 迁移说明

- **无需手动迁移。** 现有用户直接更新插件即可。
- 新阀门有合理的默认值（`Aptos Display` / `Calibri Light` / `SimHei`），无需额外配置，除非你想自定义。
- 如果你之前通过 CSS hack 设置了自定义 H1/H2 字体，现在可以改用原生阀门。
