# Deep Dive v1.1.0 版本发布说明

本次更新重点提升了 LLM 响应解析的健壮性与可靠性。通过实现多层级的 JSON 提取策略，Deep Dive 引擎现在能够处理来自更广泛模型的非规范或杂乱输出（包括参数较小或指令遵循能力较弱的模型），确保分析链条的稳定性。

## 新功能

- 🛡️ **强力 JSON 提取引擎**：
    - 引入了全新的 `_extract_json` 方法，优先识别 Markdown 代码块（支持 ` ```json ` 及原始 ` ``` ` 块）。
    - 实现了“贪婪大括号匹配”回退机制，扫描整个输出流以定位第一个 `{` 和最后一个 `}`，从而从解释性文字中精准挽救有效的 JSON 体。
- 🧹 **语法自动清洗**：
    - 增加了安全过滤层，在解析前自动识别并修复常见的 JSON 语法错误（如对象或数组末尾多余的逗号）。

## 改进点

- 🏗️ **架构重构**：
    - 将解析与提取逻辑从主 `action` 流程中完全解耦。
    - 优化了内部日志记录，更易于调试 LLM 的格式化问题。
- 🌐 **版本同步**：全站 UI 及文档显示的引擎版本号已统一同步至 v1.1.0。

## 问题修复

- 修复了当 LLM 输出多个 JSON 块或在 JSON 结构外包含解释性文本时可能导致的解析崩溃。
- 解决了微小的 JSON 语法错误会导致整个“深度下潜”过程静默失败的问题。
