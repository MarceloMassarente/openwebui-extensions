# OpenWebUI Skills Manager v0.3.2 版本发布说明

**发布日期：** 2026-05-18  
**作者：** [Fu-Jie](https://github.com/Fu-Jie/openwebui-extensions)  
**版本：** 0.3.2

---

## 最新更新

### ⚡ OpenWebUI 0.9.x 兼容性

新增运行时异步检测和兼容性适配层，支持 OpenWebUI 0.9.x 及以上版本的异步数据库模型（Skills），同时完美回退支持旧版本的同步调用。

**详细说明：** OpenWebUI 0.9.x 引入了异步/等待的数据库模型方法。本插件现已对所有 `Skills.*` 模型调用（`get_skills_by_user_id`、`update_skill_by_id`、`insert_new_skill`、`delete_skill_by_id`）封装了 `_call_openwebui_compat` 兼容层，可自动检测被调用方法是否为异步并适当等待；对于同步方法，则通过 `asyncio.to_thread` 在后台线程中运行，**无需任何手动配置**，插件在 OpenWebUI 0.9.x 前后版本均可正常工作。

### 📝 支持多行 Frontmatter 描述

`install_skill` 现在可以正确解析远程 `SKILL.md` 里的 `description: >` 和 `description: |` 块，导入后的技能描述不再被截断成单行。

**修改前：** 技能 frontmatter 的 `description:` 字段中使用 YAML 折叠式（folded）或字面式（literal）标量的多行描述无法被正确提取，导致长描述在远程技能安装时被压缩或静默丢弃。

**修改后：** Frontmatter 解析器现已正确处理 YAML folded style（`description: >`）和 literal style（`description: |`）标量，完整保留段落换行和多行内容。

### ↩️ 更稳的元数据回退

当 frontmatter 只有 `title` 没有 `name` 时，安装器会优先使用 `title`，避免退回到通用目录名。

**修改前：** 远程 `SKILL.md` frontmatter 中指定了 `title: "我的技能"` 但未提供 `name:` 时，安装器会跳过 title，回退到 URL 路径中的目录名。

**修改后：** 名称解析顺序现在会在目录/文件名回退前检查 `title`（来自 frontmatter），因此显式指定的技能标题会被正确采纳。

---

## 迁移说明

- **无需手动迁移。** 现有用户直接更新插件即可。
- 如果你运行的是 OpenWebUI < 0.9.x，插件无需任何配置变更即可继续正常工作。
- 如果你运行的是 OpenWebUI ≥ 0.9.x，异步兼容层会自动激活。
