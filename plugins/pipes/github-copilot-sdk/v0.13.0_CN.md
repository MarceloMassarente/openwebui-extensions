# GitHub Copilot SDK Pipe v0.13.0

> **发布范围**：v0.13.0 是首个同时交付 **Agent Team** 多智能体协作与完整生效的 `SESSION_MODE` 的版本。两项功能相互独立，却共同代表了 Copilot SDK Pipe 的一次质变：多个 AI 模型可以组队协作，同时对每次会话的自主程度有了真正的掌控力。

## 概览

v0.13.0 交付了两项重大新功能。**Agent Team** 引入了原生多智能体编排能力：将一组 OpenWebUI 自定义模型配置为子 Agent，指定领队，每个 Agent 自动继承与主会话相同的工具能力（OpenWebUI Skills + MCP 服务器）。**Session Mode** 则迎来了首次完整的全链路实现：`[Active Session Mode]` 指令现在会被注入到系统提示词中（与官方 Copilot SDK agent-loop 文档对齐），使 `autopilot`、`interactive` 和 `plan` 三种模式在 LLM 行为层面真正差异化——不再只是一个 Valve 配置项。

---

## 新功能

- **Agent Team**：在单次 Copilot SDK 会话中协调一组 OpenWebUI 自定义模型作为子 Agent 协同工作。通过标签（`AGENT_TEAM_TAG`）或显式模型 ID（`AGENT_TEAM_MODEL_IDS`）选择 Agent，并通过 `AGENT_TEAM_LEADER` 指定领队。每个 Agent 自动继承主会话的完整工具能力（OpenWebUI Skills + MCP 服务器），确保团队能力一致。支持全局（Valves）和按用户（User Valves）两级配置。同时修复了 `_resolve_agent_team()` 在此过程中导致的崩溃问题。

- **主动模式感知**：系统提示词的 `BASE_GUIDELINES` 之后现在会注入一个模式专属的 `[Active Session Mode]` 指令块。语言与官方 [Copilot SDK agent-loop 文档](https://github.com/github/copilot-sdk/blob/main/docs/features/agent-loop.md) 对齐：
  - `autopilot`：Agent 端到端驱动任务不中途停顿；若未调用 `task_complete`，CLI 会发送续行 nudge。
  - `interactive`：Agent 完成即时请求后停止；不注入续行 nudge。
  - `plan`：Agent 先研究并输出结构化方案，等待用户明确审批后再执行（用户的直接指令优先级最高）。

- **模式感知自适应工作风格**：`_build_adaptive_workstyle_context()` 现在接受 `session_mode` 参数，并注入模式专属前导语。每种模式不仅在模式指令层生效，还在工作风格层进行双重强化。

- **SDK 模式设置加固**：Resume 和 Create 两条会话路径均在 `asyncio.wait_for(timeout=5.0)` 内调用 `session.rpc.mode.set()`。失败时会以 `WARNING` 级别记录日志并以调试通知形式显示（即使 SDK 调用失败，系统提示词中的模式指令仍然生效）。

## 问题修复

- **新会话 `UnboundLocalError`（高危）**：create-session 路径在调试日志中引用了 `final_system_msg` 变量，但该变量只在 resume 路径中被赋值。每次全新会话的第一轮对话都会因 `UnboundLocalError` 崩溃。已通过移除该变量引用修复。

- **用户覆盖规则被静默（中危）**：`[Active Session Mode]` 指令块使用了强硬的 `MUST`/`STOP` 语言，覆盖了 `BASE_GUIDELINES §8` 中"用户的直接指令可以覆盖已配置模式"的规则。已通过在 `interactive` 和 `plan` 指令中添加 `unless the user explicitly instructs otherwise` 子句修复。

- **虚假 CLI 强制声明（中危）**：autopilot 指令无条件声称"CLI 在基础设施层面强制执行此模式"，但 `mode.set()` 可能失败。已软化为"当 SDK 级别的 autopilot 模式激活时，Copilot CLI 会自动发送续行 nudge……"——在成功和失败两种情况下均准确。


---

## 优化改进

- **系统提示词全面清理**：泛化了硬编码的 Copilot CLI 工具名（`view_file`、`show_file`、`read_bash`、`write_bash`、`mode="sync"`），移除了不适用约定（强制 `Co-authored-by: Copilot` trailer）。解决了 SQL 模式矛盾（`todos`/`todo_deps` 用于任务跟踪 vs. `interactive_controls` 用于 Rich UI 状态）。

- **SESSION_MODE 全局解析**：`effective_session_mode` 在 `_pipe_impl` 顶部统一解析，优先级为：user_valve > global valve > `"autopilot"`，与 agent team 逻辑完全解耦。

## 迁移说明

无。所有变更均为系统提示词和会话生命周期内部调整，未重命名或移除任何 Valve 配置项。
