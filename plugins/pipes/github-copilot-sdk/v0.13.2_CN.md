# v0.13.2 版本发布说明

## 概述

本补丁版本优化了 BYOK（自带密钥）模式下的模型列表管理。当用户明确指定 `BYOK_MODELS` 列表时，插件现在直接使用该列表，而不再进行不必要的 API 调用。

## 新功能

- **BYOK 模型优先级提升**：当同时配置了 `BYOK_MODELS`（显式列表）和 `BYOK_BASE_URL`（API 端点）时，插件现在优先使用用户显式配置的 `BYOK_MODELS`，直接跳过 API 获取。这既尊重了用户的明确意图，又减少了网络开销。

## 问题修复

- **移除冗余的回退逻辑**：原来插件会先尝试 API 获取，失败时才解析 `BYOK_MODELS`。这种不必要的回退链已被移除，使逻辑更清晰高效。

## 迁移说明

**对双配置模式的破坏性改动：**
- 如果你同时设置了 `BYOK_BASE_URL` 和 `BYOK_MODELS`，插件现在将直接使用 `BYOK_MODELS`，忽略 API 获取。
- 如果你想使用动态的 API 获取模型列表，请清空或移除 `BYOK_MODELS` 配置。
- 如果你希望混合模式，仅设置 `BYOK_BASE_URL`，将 `BYOK_MODELS` 留空。
