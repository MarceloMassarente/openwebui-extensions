# GitHub Copilot SDK Pipe v0.13.1

> **发布范围**：v0.13.1 是关键兼容性版本，透明支持 OpenWebUI 0.9.x 异步 API，同时完整保留对 0.9 以下（同步）版本的兼容性。未新增、移除或重命名任何 Valve 配置项。对低版本用户无任何行为变更。

---

## 概览

OpenWebUI 0.9.0 带来了全面的异步迁移：几乎所有数据库模型方法（`Files`、`Users`、`Tools`、`Models` 等）以及 `open_webui.config` 和 `open_webui.utils.tools` 中的许多辅助函数都变成了 `async def`，必须使用 `await`。插件内部调用图中存在大量直接（未 await）的调用，在 0.9.x 上会静默失效——要么返回协程对象而非预期结果，要么在线程托底路径中崩溃。

v0.13.1 没有按版本分叉代码库，而是引入了一层薄适配层，自动检测 OpenWebUI API 调用返回的是协程还是直接值，并统一处理。这样同一份代码在不同 OpenWebUI 版本下均能工作。

---

## 变更详情

- **新增兼容辅助函数**：在 `github_copilot_sdk.py` 顶部新增四个模块级辅助函数：
  - `_resolve_awaitable_compat()` — 统一解析值或协程
  - `_call_openwebui_compat()` — 异步兼容调用器，可选线程托底（用于同步 API）
  - `_resolve_awaitable_sync()` — 在同步上下文中解析协程（用于 Valve 选项提供器）
  - `_call_openwebui_sync_compat()` — 基于 `_resolve_awaitable_sync` 的同步封装
  - `_extract_search_result_items()` — 从搜索结果中提取 `.items`，兼容 list/tuple/dict/Pydantic 模型等多种形式

- **Files API 兼容**：`publish_file_from_workspace` 现使用 `_call_openwebui_compat(Files.get_file_by_id, ...)` 和 `_call_openwebui_compat(Files.insert_new_file, ...)` 替代原始 `asyncio.to_thread`，避免将异步 Files 方法传给 `to_thread` 时返回协程对象而非文件记录这一静默崩溃。

- **工具服务器配置与请求构建升级为 async**：`_read_tool_server_connections` 和 `_build_openwebui_request` 现为 `async def`。所有调用方已更新为 `await`。处理 `get_config()` 和 `Models.get_all_models()` 在 0.9.x 下的异步化。

- **工具装载链兼容**：`Users.get_user_by_id`、`Tools.get_tools_by_user_id`、`Tools.get_tool_by_id`、`get_openwebui_tools`、`get_builtin_tools` 及所有工具装载路径中的 `Models.get_model_by_id` 现通过 `_call_openwebui_compat(..., prefer_thread_for_sync=True)` 调用。

- **MCP 服务器解析升级为 async**：`_parse_mcp_servers` 现为 `async def`。异步路径中的调用方直接使用 `await`。同步上下文中的唯一调用方（`_build_session_config`）使用 `_resolve_awaitable_sync(...)` 安全桥接。

- **同步 Valve 选项提供器安全**：`Valves` 和 `UserValves` 中调用 `OWModels.search_models(...)` 的 classmethod（0.9 以下为同步，0.9.x 可能为异步）现使用 `_call_openwebui_sync_compat(...)` 替代直接调用。结果迭代使用 `_extract_search_result_items(...)` 替代 `.items` 属性访问。

- **Agent Team 模型检索兼容**：`_get_openwebui_models_by_tag` 和 `_get_openwebui_model_by_id`（async 方法）现使用 `_call_openwebui_compat(...)` 处理 `search_models` 调用。

- **系统提示词提取中的 `Models.get_model_by_id`**：`_extract_system_prompt` 现使用 `_call_openwebui_compat(...)` 处理其异步循环中的模型数据库查询。

---

## 迁移说明

- **无需任何操作**：已在 OpenWebUI < 0.9 上的用户不受影响。兼容层检测到同步行为后会像以前一样直接调用函数。
- 升级到 OpenWebUI 0.9.x 的用户无需更新插件或修改配置，Copilot SDK Pipe 即可完全正常工作。
- 未新增、移除或重命名任何 Valve 配置项。
- 所有变更均在 API 调用层内部，不影响上层业务逻辑。
